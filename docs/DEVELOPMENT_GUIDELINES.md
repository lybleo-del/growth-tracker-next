# Growth Tracker 开发规范

## 核心原则

**⚠️ 永远记住：每次代码修改，必须同时更新对应文档！**

---

## 工作流程

每次开发任务必须包含以下 3 个步骤：

### 1. 代码修改
- 完成功能开发或 bug 修复
- 确保代码质量和测试通过

### 2. 文档更新（**强制**）
根据修改范围，更新对应文档：

| 功能变更范围 | 需要更新的文档 |
|------------|--------------|
| 新功能/功能变更 | `prd.md` (产品需求文档) |
| 技术架构变更 | `arch.md` (技术架构文档) |
| 重要的 UI/UX 变更 | `2026-05-26-growth-tracker-nextjs-redesign.md` (设计文档) |
| 数据模型变更 | `schema.sql` (数据库 schema) + `types.ts` (类型定义) |

### 3. 提交和推送
- 提交信息要清晰说明代码变更和文档更新
- 推送到仓库

---

## 文档位置

所有文档位于 `/docs/` 目录下：

```
/docs/
├── prd.md                          # 产品需求文档 (PRD)
├── arch.md                         # 技术架构文档
├── 2026-05-26-growth-tracker-nextjs-redesign.md  # 详细设计规范
├── 2026-03-17-growth-tracker-design.md           # 原始设计文档
└── DEVELOPMENT_GUIDELINES.md       # 本开发规范 (你正在看的)
```

## 检查清单

在完成任何开发任务后，请确认：

- [ ] 代码功能已实现/修复
- [ ] 代码质量符合要求
- [ ] 已更新对应文档 (PRD/架构/设计文档)
- [ ] 已提交到 Git
